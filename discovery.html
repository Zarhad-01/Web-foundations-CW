Week 1 - Welcome to WebF1

First entry on my discovery log. This is to record what I learn (and how to fix any problems I encounter) every week of doing WebF1. This week we learnt to run and setup the VM, which will be used to host our websites. We briefly looked at the commands we would be using to host our website.

  1. Use WinSCP to upload the website (ensure homepage is called index.html).
  2. Go to port.ac.uk/myvm and run the vm. and launch a connection using the Chrome Secure Shell plugin.
  3. Move into directory using the “cd Website_File” command.
  4. Host the website using the “http-server” command.
  5. Ensure you are at the uni or using the vpn.
  6. Put the IP address of the VM into your URL and the website should be there.

Week 2 - Web Security

This week we learnt about the differences between secret, sensitive and public information. Secret information is a type that shouldn't be able to be accessed by anyone. Sensitive information should only be able to be accessed only when required by authorisation of the user. Public information is something that the user has deemed acceptable to be viewed by anyone. We discussed the issue surrounding social media users sharing sensitive information that could be used to build a profile on them and then eventually hacked.

Our lecture this week was about information and social engineering and was led by Nick savage. He talked about how small amounts of personal information could be used against the user. We watched a video of a woman extracting information from a telephone company about the interviewer using various social engineering techniques.

Week 15 - Standards

This week we were discussing the different standardisation bodies and how now having standardisation can cause issues and examples of when this has occurred. One example of this were how the west and east of the USA used different width rail tracks for trains. This meant that trains from the west couldn’t go east and if people or cargo wanted to it would have had to stop and be loaded onto different trains.

Week 16 - Usability

This week we discussed the topic of useability and the factors attributed to this, examples of this are:

  • Target audience of the website.
  • The reason they are visiting your website.
  • What pages will be the most used.
  • How simple the user interface be.

The user interface is one of the more important features because it essentially the face of your website, and who doesn’t love a good looking face, eh?
